What changes

  • Main area gets real width for tables/editors.
  • Jobs render incrementally and show filtered history.
  • Wiki/source lists use search + paging/virtual windows.
  • Tables get stable min widths and horizontal scroll only when needed.

Trade-offs

  • More JS structure than today, though still no build step.
  • Some existing snapshot-style static tests need updates.
  • Large lists become paged/windowed, so “all rows in DOM” is gone by design.
